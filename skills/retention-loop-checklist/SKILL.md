# Retention loop checklist

## Objetivo

Validar **por qué alguien volvería** y qué falta para que el loop sea creíble.

## Checklist

| Área | Pregunta | Notas / acción |
|------|----------|----------------|
| Core loop | ¿El loop de una sesión es satisfactorio en el tiempo objetivo (ej. menores de 15 min)? | |
| Meta a corto plazo | ¿Hay objetivo claro en la primera hora? | |
| Progresión | ¿Sensación de avance (skill, desbloqueo, historia)? | |
| Contenido | ¿Hay variación entre sesiones o todo idéntico? | |
| Social | ¿Depende de multiplayer? ¿charter lo permite? | |
| Fricción | Login, tutoriales largos, crashes, tiempos de carga | |
| Motivos de retorno | Eventos, daily, ranking, historia, craft — marcar cuáles son reales vs idea |
| Fatiga | ¿Demasiada presión FOMO para el público objetivo? | |

## Salida

- **Top 3 palancas** de retención viables en la fase actual.
- **Top 3 riesgos** de churn.
- **Una** recomendación priorizada para el siguiente brief (Gameplay/World/LiveOps).

## Reglas

- No inventar sistemas de temporada completos si el slice es single mission; marcar **placeholder de diseño**.
