# Benchmark summary

## Objetivo

Tabla **comparable** entre referencias para que Camilo y diseño vean **dónde jugar** y **dónde no competir**.

## Salida

1. **Lista de referencias** (3–7): nombre + por qué está en la lista.
2. **Ejes de comparación** (elegir 5–8 y mantenerlos fijos en todo el doc):
   - Core loop (1 línea cada una)
   - Sesión / duración típica percibida
   - Dificultad / curva
   - Monetización visible (si aplica)
   - Identidad audiovisual (nivel, no juicio de gusto)
   - Features técnicas notables (co-op, construcción, etc.)
3. **Matriz** referencia × eje (breve texto o símbolo: ✓ / — / ~).
4. **Implicaciones para nuestro slice**: 3–5 bullets accionables.
5. **Anti-benchmark**: qué **no** copiar y por qué (riesgo o identidad).

## Reglas

- Evitar párrafos largos por título; preferir matriz + bullets.
- Si falta acceso al juego, marcar **percepción externa** (trailer/review).
