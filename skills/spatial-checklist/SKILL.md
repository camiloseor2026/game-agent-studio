# Spatial checklist

## Objetivo

Cierre rápido de **lectura espacial**: el jugador entiende dónde está, a dónde ir y a qué escala juega.

## Checklist (marcar OK / problema / N/A)

| Criterio | Notas |
|----------|--------|
| **Escala humana** | Props o módulos que anclan tamaño (puerta, barandilla). |
| **Landmarks** | Al menos un punto reconocible por zona desde distancia media. |
| **Horizonte / fondo** | No confundir skybox con suelo caminable. |
| **Rutas legibles** | Camino principal perceptible sin UI de flechas (silueta, luz, apertura). |
| **Cobertura** | Si el combate lo exige: volumetría de cobertura a altura de rodilla/pecho; no solo “cajas al azar”. |
| **Líneas de vista** | Sniping o engagement largo: ¿intencional o sorpresa no deseada? |
| **Audio/espacio** | Zonas acústicas o vacías que puedan romper feedback (nota breve). |
| **Objetivo visible** | En slice: ¿el jugador puede inferir el siguiente hito espacial? |

## Salida

1. Resumen **pass / pass con deuda / fail** con 3 bullets máx.
2. **Deuda espacial** priorizada para siguiente iteración.
3. **Handoff Art**: qué debe respetarse vs qué es provisional en greybox.
