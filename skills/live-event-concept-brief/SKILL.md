# Live event concept brief

## Objetivo

Un **evento vivo** descrito lo suficiente para que producto y diseño decidan sí/no y qué implementar después.

## Plantilla

```markdown
# Event concept — [nombre] — [ventana temporal aproximada]

## Objetivo de negocio / producto
- …

## Jugador: promesa en una frase
- …

## Duración y cadencia
- Inicio / fin (relativo o absoluto TBD)
- ¿Se repite? frecuencia sugerida

## Contenido / hooks
- Gameplay: …
- World/Art: …
- Comunicación externa (si aplica): …

## Requisitos técnicos (alto nivel)
- Servidor obligatorio: sí/no
- Datos remotos: sí/no
- Feature flags: sí/no

## Riesgos
- Scope creep, fatiga, pay-to-win percibido, etc.

## Señales de éxito (cualitativas o métricas conceptuales)
- …

## Fuera de alcance en esta fase
- …
```

## Reglas

- Si el charter excluye multiplayer/live backend, marcar **solo diseño aspiracional** o **single-player façade**.
